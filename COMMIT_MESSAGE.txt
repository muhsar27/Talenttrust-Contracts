fix: resolve compilation errors by refactoring contractimpl macro usage

Fixes 8 compilation errors (E0425) caused by multiple #[contractimpl] 
blocks across module files trying to reference generated types that only 
exist in the main contract.

Changes:
- Remove #[contractimpl] from create_contract.rs, deposit.rs, finalize.rs, 
  and migration.rs modules
- Convert module methods to standalone implementation functions (_impl suffix)
- Add entrypoint wrappers in lib.rs #[contractimpl] block that delegate 
  to module functions
- Clean up unused Escrow imports from module files
- Make helper functions pub(crate) for cross-module access

This approach maintains modular code organization while respecting 
Soroban's single-contractimpl constraint.

Verified:
- cargo check passes with no errors
- All 8 E0425 errors resolved
- Dispute resolution entrypoints (raise_dispute, resolve_dispute) remain 
  functional
